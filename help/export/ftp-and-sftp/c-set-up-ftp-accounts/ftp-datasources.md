# FTP ベースのデータソース

Analyticsを使用して、FTP ベースのデータソースを作成および管理できます。このデータソースでは、FTP ファイル転送を利用して、オフラインまたは過去のデータをCX Enterpriseに読み込みます。

データソースインスタンスを作成した後、ツールは、データソースファイルのアップロードに使用できるFTPの場所を提供します。 アップロードが完了すると、データソースは自動的にそれらを検出して処理します。 ファイルが処理されると、データはAnalytics レポートで使用できるようになります。

データソースマネージャーの「[!UICONTROL 作成]」タブでは、選択したレポートスイートに新しいデータソースインスタンスを設定できます。 データソースウィザードは、データソーステンプレートの作成プロセスをガイドし、データをアップロードするためのFTPの場所を作成します。

「[!UICONTROL  データソースを管理]」ウィンドウで、データソースを見つけ、「FTP情報」リンクを選択します。 FTP ログイン情報は、[!UICONTROL  アップロード/FTP情報] セクションの[!UICONTROL  データ Sourceの有効化] ウィンドウに表示されます。

FTP制限とデータ保持について詳しくは、[FTP制限とデータ保持](/help/export/ftp-and-sftp/ftp-limits.md)を参照してください。

## 分類およびデータソースのアップロードに必要な .fin ファイルについて {#section_1484719F8A134EAE91212DBD8F15174F}

分類またはデータソースファイル （`.tab`または`.txt`）をアップロードする場合、アップロードには、インポートするデータファイルとまったく同じ名前で、`.fin`拡張子の空のファイルをアップロードする必要もあります。 この `.fin` ファイルは、完了通知ファイルです。 このファイルは、データファイルが FTP アカウントに完全にアップロードされたことをシステムに通知します。 `.fin` ファイルがアップロードされることで、お客様が読み込みデータのアップロードを完了したことをアドビで認識できます。 このファイルが送信された後、アドビでは両方のファイルを FTP から削除し、読み込みの処理を開始します。

ファイルの読み込み：`Classifications.tab`

ファイルの終了：`Classifications.fin`

データソースまたは SAINT ファイルをアップロードしても、`.fin` ファイルをアップロードしないと、処理のためのキューに追加されません。 ファイルはFTP上に残り、CX Enterpriseのデータには適用されません。 この通知は、レポートの[!UICONTROL FTP アカウントの作成] ウィンドウで[!UICONTROL 通知受信者]としてメールアドレスを入力した場合にのみ通知されます。 このフィールドにメールアドレスが入力されていない場合、通知は送信されません。

ファイルをアップロードし、`.fin` ファイルをアップロードしたが、ファイルにエラーがある場合、ファイルは処理のために送信されますが、エラーによって処理が中止され、ファイルはエラーフォルダーに送信されます。 この場合、[!UICONTROL FTP アカウントを作成]ウィンドウで「[!UICONTROL 通知受信者]」フィールドに指定された電子メールアドレスに通知が送信されます。 メールアドレスが入力されていない場合、通知は送信されません。
