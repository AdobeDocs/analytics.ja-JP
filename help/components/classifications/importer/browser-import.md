# ブラウザー読み込み（レガシー）

{{classification-importer-deprecation}}

ブラウザーを使用して、分類データを読み込み（アップロード）できます。 この方法では、分類データを1つのレポートスイートにアップロードできます

## ブラウザーインポート {#concept_314FB3D5FD5A439B9CFEDE37A3337BA7}

ブラウザーを使用して、分類データを読み込み（アップロード）できます。 この方法では、分類データを1つのレポートスイートにアップロードできます

**[!UICONTROL 管理者]**／**[!UICONTROL 分類インポーター]**

## 分類ブラウザーインポート - フィールドの説明 {#section_F628C47081DA4026A4D30E3D3454B1DA}

| 要素 | 説明 |
| --- | --- |
| レポートスイートの選択 | 分類データを読み込むレポートスイート。 読み込みデータファイルは、レポートスイートのデータセットの形式と一致する必要があります。 |
| 分類されるデータセット | 分類を受け取るデータセット。 ドロップダウンリストには、分類用に設定されたレポートスイート内のすべてのレポートが含まれます。 |
| 読み込むファイルを選択 | アップロードするインポートデータファイルを参照して指定します。  注意：アップロードできるファイルのサイズは最大で 1 MB です。 |
| 競合に関するデータの上書き | 読み込まれたデータと競合する既存のデータが自動的に上書きされます。<br>**重要**：このオプションは、新規分類アーキテクチャで有効になっているレポートスイートでは使用できません。 |
| 読み込み完了後に分類ファイルを自動的にダウンロード | 新しくアップロードされた分類データを含むデータセットを表すタブ区切りファイルを自動的にダウンロードします。 インポートによって一意のIDが作成された場合、またはエラーが発生した場合、Adobeはこのファイルを自動生成します。<br>**重要**：このオプションは、新規分類アーキテクチャで有効になっているレポートスイートでは使用できません。 |


## ブラウザーを使用した分類のインポート {#task_D7D51CB6FB35437AB68599B1B23FEAC1}

1. **[!UICONTROL 管理者]**／**[!UICONTROL 分類インポーター]**&#x200B;をクリックします。
1. 「**[!UICONTROL ファイルのインポート]**」をクリックします。
1. 「**[!UICONTROL ブラウザーインポート]**」フィールドを設定します。
1. 「**[!UICONTROL ファイルのインポート]**」をクリックします。
1. ステータスウィンドウで、処理メッセージを確認します。
1. （任意）「**[!UICONTROL アップロードの完了後、自動的に分類ファイルをダウンロードする]**」を選択した場合は、処理完了時に結果ファイルを保存する場所を指定します。 このオプションは、新しい分類アーキテクチャで有効なレポートスイートには使用できません。

>読み込みが成功すると、書き出しの適切な変更が直ちに表示されます。 ただし、ブラウザーの読み込みを使用する場合は最大4時間、FTPの読み込みを使用する場合は最大24時間かかります。
