# 仮想レポートスイートのワークフロー

仮想レポートスイートの実装を計画するために必要な手順の概要です。

![](/help/admin/tools/manage-rs/edit-settings/general/c-server-side-forwarding/assets/step1_icon.png)仮想レポートスイートの戦略を策定します。

戦略を決定する際は、[グローバル／マルチスイートタギングの考慮事項](/help/components/vrs/vrs-considerations.md)とよくある質問を確認してください。

![](/help/admin/tools/manage-rs/edit-settings/general/c-server-side-forwarding/assets/step2_icon.png)親レポートスイートに 1 つまたは複数のセグメントを適用して[仮想レポートスイートを作成](/help/components/vrs/c-workflow-vrs/vrs-create.md)します。

仮想レポートスイートは、**[!UICONTROL コンポーネント]**/**[!UICONTROL 仮想レポートスイート]**&#x200B;を通じて利用できる仮想レポートスイートマネージャーで作成されます。

作成された仮想レポートスイートは、UI全体で基本レポートスイートと同じように扱われ、ほとんどの拡張機能でサポートされます。 例えば、仮想レポートスイートはレポートスイートセレクターに表示され、他のレポートスイートと同様に個別に選択できます。

レポートスイート名の横にある情報アイコンをクリックして、[それについての情報を表示します](/help/components/vrs/c-workflow-vrs/vrs-view.md)。

![](/help/admin/tools/manage-rs/edit-settings/general/c-server-side-forwarding/assets/step3_icon.png)データをセグメント表示できるよう、プロジェクトまたはレポートを開いて仮想レポートスイートを選択します。

![](assets/step4_icon.png) [&#x200B; 仮想レポートスイートを管理](/help/components/vrs/c-workflow-vrs/vrs-manage.md)します。

仮想レポートスイートマネージャーを使用すると、仮想レポートスイートを編集、追加、タグ付け、削除、名前変更、承認、コピー、書き出し、フィルターできます。 管理者以外のユーザーには表示されません。
