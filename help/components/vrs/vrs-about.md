# 仮想レポートスイートの概要

仮想レポートスイートを使用すると、Adobe Analytics データをセグメントに分割して、各セグメントへのアクセスを制御できます。

多くの顧客は、グローバルレポートスイートにデータを取り込むだけでなく、小規模なレポートスイートにデータを取り込みます。 変数を複数のレポートスイートに設定し、そのデータを複数のレポートスイートに送信します。 これは、 *マルチスイートタグ付け* または *ベース／親レポートスイート* と呼ばれます。

例えば、あらゆるデータを1つのレポートスイートで収集する場合、セカンダリレポートスイートを設定することで、社内の他のメンバーがそのデータの一部にアクセスできるようにしますが、そのすべてをアクセスできるわけではありません。 データは地域ごとに分けられるかもしれません。 国ごとに異なるweb サイトを運営していることもあります。 たとえば、大企業に属する特定のブランドが、それぞれ独自のマーケティングチームを持っているとします。

*仮想レポートスイート*&#x200B;を使用すると、複数のレポートスイートの代わりにセグメントを使用してこの分岐概念を再現できます。 データは、1 つのレポートスイートに送信された後、セグメントに従って分割されます。 複数ブランドの例を使用すると、アイテムが属するブランドのpropを設定できます。 セグメントを使用して、各propに割り当てられたアイテムをレポートできます。 これらのセグメントはそれぞれ独自のビューとなり、新しいレポートスイートが作成されます。 特定のセグメントにデータを送信することはなく、グローバルレポートスイートにのみ送信しますが、異なるレポートスイートであるかのようにレポートで機能します。

バーチャルレポートスイートは、eVarの設定、処理ルール、分類など、基本レポートスイートのサービスレベルのほとんどを継承します。次の設定は継承されません。

* レポートスイート ID （RSID）
* レポートスイート名
* 権限グループ（仮想レポートスイートは、独自の権限グループに割り当てることができます）

## 仮想レポートスイートの利点 {#section_3420422FE6DF46EAB151FD9442AAFDC4}

顧客はセカンダリサーバーコールに料金を支払うため、こうしたコールを排除すると、大幅な節約につながります。 バーチャルレポートスイートも完全に遡及的です。 グローバルレポートスイートに既にデータが含まれている場合、関連データは新しい仮想レポートスイートに自動的に含まれます。 新しいセカンダリレポートスイートでは、作成後にのみデータの収集が開始されるため、過去のデータは含まれません。 Analyticsを実装する場合、グローバルレポートスイートと各二次レポートスイートの実装を作成するのではなく、1つのレポートスイートにデータを送信するだけで済みます。

仮想レポートスイートのヘルプ：

* すべてのサイト/ドメインで単一のレポートスイート ID （RSID）を使用できるようにすることで、実装を簡素化します。 あらゆるデータを単一のレポートスイートに集約することで、次世代のAdobe Analyticsに向けて、顧客分析を強化できます」と、Kristenは述べています。
* 組織内のビジネスユーザーは、常に関連するデータセグメントのみを表示します。
* 管理者ユーザーが実装後にデータアクセスをより簡単かつ詳細に制御できるようにすることで、セキュリティを向上させます。
* 人物指標
* データの単一顧客像（今後）
* データをセグメンテーションするための無制限の仮想レポートスイートを作成する機能

## 仮想レポートスイートの制限 {#section_F22A6DEBDC9848429E446F4CC2C4EEDE}

仮想レポートスイートには次の制限があります。

* また、セグメントのすべての制限が仮想レポートスイートに適用されます。

  仮想レポートスイートは、レポートスイートに適用されるセグメントに過ぎません。 各レポートスイートは独自の Data Warehouse と独自のデータフィードを持つので、複数のレポートスイートを使用すると、セグメントでは提供されない利点が得られます。
* リアルタイムレポート
* 設定と変数名は、完全なレポートスイートのようにカスタマイズすることはできません。

## 仮想レポートスイートとマルチスイートタグ付けの組み合わせ {#section_026FA3FCD7314DD18220E73EC5702AFF}

場合によっては、仮想レポートスイートとマルチスイートタグ付けの両方を使用する利点があります。

例えば、retailerでは、企業ごとにレポートスイートを利用し、企業ごとにバーチャルレポートスイートを利用して、データを地域別に分析できます。 同様に、スポーツリーグでは、各チームにレポートスイートを使用し、その後、バーチャルレポートスイートを使用して、チームの地域のファンと地域外のファンを分けることができます。
