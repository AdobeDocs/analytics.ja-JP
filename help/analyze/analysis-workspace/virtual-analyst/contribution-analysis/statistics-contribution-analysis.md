# 貢献度分析で使用される統計的手法

貢献度分析は、Adobe Analytics で異常値と見なされた結果に貢献した要因を発見するために設計された、集中的な機械学習プロセスです。目的は、ユーザーが、別の方法よりもずっと迅速に、焦点を当てるべき領域や追加の分析のオポチュニティを見つけるのを支援することです。

貢献度分析は、ユーザーの貢献度分析レポートで使用できるすべてのディメンション項目に2つのアルゴリズムを実行することで、これを実現します。 アルゴリズムは、次の順番で動作します。

1. 各ディメンションに対して、クラメールのV検定統計を計算します。 次の例では、2 つの期間にわたる国別ページビュー数の分割表について検討します。

   ![](assets/contingency_table.png)

   表1では、クラメールのVを使用して、期間1（例：履歴）と期間2（例：異常値が発生した日）の国別ページビュー数間の関連を測定できます。 クラメールのVの低い値は、低いレベルの関連性を意味します。 クラメールのVの範囲は0（関連なし）～ 1（完全な関連付け）です。 クラメールのV統計は、次のように計算できます。

   ![](assets/cramers-v.png)

1. ディメンション項目ごとに、個人の残差(PR)を使用して、異常な指標と各ディメンション項目の間の関連性を測定します。 PR は、標準正規分布に従い、偏差が比較可能でなくてもアルゴリズムで 2 つのランダム変数の PR を比較できます。実際には、エラーは不明で、有限のサンプル補正を使用して見積もられます。

   最終的な結果を得るには、各ディメンション項目のスコアがクラメールのV指標で重み付けされ、0 ～ 1の数値に再スケールされ、貢献度スコアが提供されます。
