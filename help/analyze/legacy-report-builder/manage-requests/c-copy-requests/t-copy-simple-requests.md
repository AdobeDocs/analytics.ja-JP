# 単純なリクエストのコピー

{{legacy-arb}}

参照型リクエストではなく、単純なリクエストをコピーします。 単純な要求とは、別の要求への参照やセルの内容を含まない要求です。

データフィルター、[相関フィルター](/help/analyze/legacy-report-builder/manage-requests/c-copy-requests/t-copy-referential-requests.md)などのパラメーターの入力としてセルの値を使用できます。 これらのフィルターは、マッチングまたはトレンドを使用し、以前のリクエストの結果または入力セルと呼ばれるセルのユーザー入力コンテンツに基づいています。

シンプルなリクエストをコピーするには

1. 有効なリクエストを作成します。
1. リクエストがマッピングされているセルの 1 つを右クリックするか、リクエストを含むセルの領域を選択します。

   リクエストでカバーされるセルのグループからコピーするセルを選択する際に、一貫性を持たせます。 好ましい選択肢は、リクエストでカバーされる一連のセルの先頭と左のセルで、左から右に作業します。 これは、Excel スプレッドシートに数百の列と数千の行が右方向および下方向に拡張できるためです。 リクエストに関連付けられたセルのセットの一番右または一番下のセルからリクエストコピーを開始する場合、貼り付けるセルがスプレッドシートの左端または上端を超えて広がる場合、システムはリクエストを貼り付けることはできません。
1. 「**[!UICONTROL リクエストをコピー]**」を選択します。
1. スプレッドシートの別の部分で、空のセル（リクエストを含まないセル）を右クリックします。

   既に作成したリクエストを失ったり破損したりするのを防ぐために、リクエストを含むセルを現在リクエストにマッピングされているセルに貼り付けることはできません。 リクエストを含むセルをコピーまたはカットする場合、リクエストを含むセル（またはセルのセット）を右クリックしても、ショートカットメニューで「[!UICONTROL  リクエストをペースト ]」オプションは使用できません。 貼り付け操作の宛先として別のセルを選択して、リクエストが重複しないようにします。 これは、貼り付けるリクエストを持つ単一のセルを選択するか、リクエストを含むセルの領域を選択するかに適用されます。
1. 「**[!UICONTROL リクエストを貼り付け]**」をクリックします。

   オリジナルのリクエストのコピーは、セル内のオリジナルのリクエストを基準にした位置（複数可）に配置されます。

   >[!NOTE]
   >
   >コピーされるのはリクエストのみで、セルの内容はコピーされません。 リクエストに基づかず、セルに表示されるデータ（テーブル列ヘッダーや行識別子など）を理解するために関連するその他の情報がある場合は、Excelの標準コマンド「コピー&amp;ペースト」を使用します。

   Excelでは、セルの内容のコピーとリクエストのコピーに異なるクリップボードを使用するため、コピー/ペースト要求とコピー/ペースト要求を連続して実行することで、リクエスト以外のセルの内容とリクエストの両方をコピーできます。 ただし、スプレッドシートでリクエストに書式を適用してからコピー&amp;ペーストすると、Report Builderは元の書式（枠線、フォントなど）を再現します。 貼り付け領域で使用します。

   リクエストをペーストする前に、クリップボードにコピーまたはカットしたリクエストを変更すると、クリップボードからリクエストが削除されます。 したがって、リクエストを元の状態に保つために、リクエストをコピーしてから貼り付けるまでの間にリクエストを変更しないでください。
