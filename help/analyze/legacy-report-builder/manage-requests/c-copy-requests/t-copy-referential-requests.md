# 参照リクエストのコピー

{{legacy-arb}}

データフィルター、相関フィルターなどのパラメーターの入力としてセルの値を使用できます。

スプレッドシートで参照要求を伝播またはコピーして貼り付けるには、次の手順を実行します。

* スプレッドシートに少なくとも1つの有効なリクエストを作成する必要があります。

* リクエストによって生成されるデータには、値が（分類または一致するフィルターを使用して）別のセルのリクエストに依存するか、セルに入力されたデータから入力を取得するフィルターに依存するセルが含まれている必要があります。

異なるワークシート内のリクエストから入力フィルターを参照するリクエストを作成することもできますが、異なるワークブックを作成することはできません。 例えば、シート 2のリクエストでは、シート 1の特定のセルからのレポートスイートと、シート 2のリクエストのセルからの日付範囲を使用できます。 新しい出力は、同じワークブック内のシートまたは新しいシートのいずれかに配置できます。 相対リクエストをペーストすると、入力フィルターが、コピーされたリクエスト出力が配置されているワークシートとは異なるワークシート上にある場合、そのフィルターは絶対フィルターとしてペーストされます。

>[!NOTE]
>
>複数のワークシートで1つのリクエストを出力することはできません。 さらに、コピーしたリクエストの一部を新しいワークブックに貼り付けることはできません。これは、リクエストに他のワークシートからの入力フィルターが含まれているためです。 入力フィルターには、セルのレポートスイート、セルの日付範囲、セルのフィルター、その他の関連パラメーターが含まれます。

**参照要求をコピーするには**

1. コピーするリクエストを含んだセルを、入力セルまたは参照先セルを含めて選択します。
1. 強調表示されたセル内を右クリックして、ショートカットメニューから「**[!UICONTROL リクエストをコピー]**」を選択します。

   リクエストおよび入力セルがある領域を選択すると、それらの要素を含んだセルが強調表示されます。
1. リクエストの貼り付け先として、1 つのセルまたは連続するセル範囲を選択します。

   他のデータまたはリクエストが含まれているセルやセル範囲を選択しないように注意してください。
1. 選択した単一のセルまたはセル範囲の左上隅にあるセルを右クリックし、「**[!UICONTROL リクエストを貼り付け]**」を選択します。

   入力セルを含むリクエストをペーストする場合、[!UICONTROL  リクエストをペースト ]の下のオプションには次のものが含まれます。

   **絶対入力セルを使用：**&#x200B;選択したセルに関連付けられているリクエストと書式のコピーを、ハイライト表示したペースト領域にペーストします。 入力セル（元のリクエストのいずれかで参照されているセル）はペーストされません。 代わりに、入力セルは以前と同じ位置に残ります。

   **相対入力セルを使用：**&#x200B;選択したセルに関連付けられているリクエストと書式のコピーを、入力セルのコピーを含め、ハイライト表示したペースト領域にペーストします。 入力セルに対するリクエストの空間的な関係は、元のリクエストと同じです。 ただし、新しく貼り付けられたセルにはリクエストのコピーが含まれていますが、最初はコンテンツがありません。 これは、ペースト操作で入力セルが再作成された場合、入力セルにデータが関連付けられないためです。 新しく貼り付けられたリクエストのデータを表示するには、入力セルに値を入力し、リクエストを更新する必要があります。
