# 異常値検出{#anomaly-detection}

{{legacy-arb}}

異常値検出は、統計的なモデリングを使用して、データ内の予期しないトレンドを自動的に見つけます。 モデルによって指標を分析し、値の下限、上限、予想される範囲を決定します。 予期しないスパイクまたは下落が発生した場合にレポートします。

調査できる異常値の例を次に示します。

* 平均注文額の急激な下落
* 売上高の低い注文数のスパイク
* 体験版登録数のスパイクまたは下落
* ランディングページビュー数の下落
* ビデオバッファーイベントのスパイス
* 低ビデオビットレートのスパイク

>[!NOTE]
>
>異常値検出は、「日」の精度を選択した場合にのみ利用できます。

レポートビルダーは、これらの値を選択した指標に適用します。 例えば、ページビュー指標を選択して異常値検出を適用すると、*`Page Views Lower Bound`*&#x200B;指標が使用されます。

**異常値検出の計算方法**

異常値検出では、トレーニング期間を使用して、1日あたりの予測間隔データを計算、学習、レポートします。 トレーニング期間とは、正常な期間と異常な期間を特定し、学習した内容をレポート期間に適用する過去の期間です。 マーケティングレポートでは、30、60、90のトレーニング期間を利用できます。 Report Builderでは、30日間ご利用いただけます。

トレーニング期間は、選択したレポート期間と必ずしも同じではありません。 レポートグラフには、カレンダーで指定した日付範囲の期間が表示されます。

データを計算するために、各指標の1日の合計は、次の各アルゴリズムを使用してトレーニング期間と比較されます。

* Holt Winters Multiplicative （トリプル指数平滑法）
* Holt Winters Additive （トリプル指数平滑法）
* Holts トレンドの修正（二重指数平滑化）

各アルゴリズムは、SSE （Sum of Squared Errors）が最も小さいアルゴリズムを決定するために適用されます。 次に、平均絶対誤差（MAPE）と現在の標準誤差を計算して、モデルが統計的に有効であることを確認します。

これらのアルゴリズムは、将来の期間における指標を予測するために拡張することができます。

トレーニング期間はレポート期間の開始日によって異なるため、同じ日付にレポートされたデータに2つの異なる期間の一部として違いが表示される場合があります。

たとえば、1月1～14日のレポートを実行し、1月7～21日のレポートを実行する場合、2つの異なるレポートで1月7～14日の間に同じ指標に対して異なる予測データが表示される場合があります。 これは、トレーニング期間の違いの結果です。

| レポート範囲 | トレーニング期間 |
|--- |--- |
| 1月1～14日 | 11月27日～12月31日 |
| 1月7、21日 | 12月4日～1月6日 |
