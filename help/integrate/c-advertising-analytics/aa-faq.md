# よくある質問

## アクセス／権利 {#access}

+++ この機能にアクセスするには、Adobe Advertising CloudまたはAdobe Advertising Cloud(AMO) の顧客である必要がありますか？

いいえ。この機能は、Advertising Cloud以外のお客様と AMO 以外のお客様で利用できます。 </p> <p>AMO のお客様は、既存の Analytics-AMO 統合を利用できますが、Ad Analytics は使用できません。

+++

+++ Advertising Analyticsの使用権限を得られるのは、どのAdobe Analytics SKU ですか？

Advertising AnalyticsはAdobe Analyticsで使用できます

* [Select](https://www.adobe.com/jp/data-analytics-cloud/analytics/select.html)

* [Prime](https://www.adobe.com/jp/data-analytics-cloud/analytics/prime.html)

* [Ultimate](https://www.adobe.com/jp/data-analytics-cloud/analytics/ultimate.html)

+++

+++ Advertising Analyticsを使うには、追加料金を支払う必要がありますか？

いいえ。適切な SKU プロビジョニング以外では、Advertising Analyticsに追加費用は発生しません。

+++

+++ Advertising Analyticsはサーバーコールの使用状況に対してカウントされますか。

いいえ、Advertising Analyticsでは、サーバーコールの費用が発生しない特別なデータソースタイプを使用します。

+++

+++ 既にAdvertising Cloud/AMO を使用している場合、Advertising Analytics機能を引き続き使用できますか。

互換性のある検索エンジンアカウントはAdvertising Analyticsに渡され、読み取り専用として表示されます。 すべての編集または更新は、Advertising Cloud／AMO で処理されます。

+++

+++ 正しい SKU を所有していますが、Advertising Analyticsにアクセスできません。なぜですか。

Advertising Analyticsは、Adobe Analytics管理者のみが使用できます。ただし、管理者は管理者以外のユーザーにアクセス権を付与することができます。 アクセス権限について管理者まで問い合わせてください。

+++

## Advertising Analytics の使用 {#using}

+++ Advertising Analyticsには、どの検索エンジンアカウントが含まれますか？

検索エンジンのアカウントには、Google AdWords とMicrosoft Bing が含まれます。

+++

+++ Advertising Analyticsにアクセスするには、どこに移動しますか？

Adobe Analyticsにログインした後、 [!UICONTROL 管理者]. 次に、 [!UICONTROL Advertising Analytics] をクリックして検索エンジンアカウントを追加します。

+++

+++ 収集され、Analytics に渡されるデータはどのようにしてですか？

Advertising Analyticsは、一連のカスタム API を使用して、Adobe Advertising Cloudを介して検索エンジンから Analytics にデータを渡します。

+++

+++ この統合で得られる検索データは何ですか？

次の情報が得られます。

* 印象
* クリック数
* コスト
* 品質スコア
* 検索エンジンから直接の平均順位
* AMO ID インスタンス（「インスタンス」をクリック）。

+++

+++ 他の Analytics データ（指標/ディメンション）でAdvertising Analyticsデータを分類できますか？

いいえ。生の検索データは、独立したデータセットとして取り込まれます。 ただし、他の Analytics データによって内訳を示すことができる、クリックデータのインスタンスバージョンがあります。

+++ 自分のアカウントに関する様々なステータスインジケーター（保留中、アクティブ、一時停止など）の定義は何ですか？ これらの各ステータスインジケータは、各検索エンジンアカウントのライフサイクルステージを識別します。

* [!UICONTROL Pending]
* [!UICONTROL 一時停止]は、以前にアカウントが設定されているが、アクティブでない状態になっていることを表します。
* [!UICONTROL アクティブ]は、アカウントが完全に設定済みで、検索データを取り込んでいることを表します。

+++

+++ Advertising Analyticsアカウントを特定のレポートスイートにマッピングしようとしていますが、レポートスイートモーダルでは使用できません。 なぜですか？

レポートスイートをAdvertising Analyticsアカウントに割り当てる前に、目的のレポートスイートが [Advertising Analyticsレポート用にプロビジョニングされました](/help/integrate/c-advertising-analytics/c-adanalytics-workflow/aa-provision-rs.md)
これは別の管理者ページでおこないます。このページには、管理者/レポートスイート/からアクセスできます。 [レポートスイートを選択] /設定を編集/ Advertising Analytics設定

+++

+++ 仮想レポートスイートをAdvertising Analyticsアカウントに割り当てることはできますか？

仮想レポートスイートではデータが収集されないので、Advertising Analyticsアカウントを仮想レポートスイートに直接マッピングすることはできません。 ただし、データを表示する仮想レポートスイートの親レポートスイートにAdvertising Analyticsアカウントをマッピングすることはできます。 AMO ID（またはその分類）に基づくセグメントロジックに「or」条件を含めない限り、検索エンジンの指標（クリック、コスト、インプレッション）が仮想レポートスイートに表示されない場合があります。 例：「AMO ID が存在するすべてのヒット」を追加すると、セグメント内の検索エンジンの指標が含まれます。

+++

+++ Advertising Analytics指標は <b>マーケティングチャネル</b> レポート？

いいえ。これらはマーケティングチャネルレポートには含まれません。

+++

+++ 質問：検索データは Analytics にいつ取り込まれますか。

回答：検索データは、Adobe Analytics データセンターのタイムゾーンの午前 6 時ごろに検索エンジンから取り込まれます。この時点で AMO データが収集され、レポートスイートに挿入されます。さらに AMO データは Analytics へのデータ挿入の一環としてレポートスイートのタイムゾーンに変換されます。

+++

+++ 以下が可能です。 <b>クリック前にキャプチャされました</b>? インプレッション数、コスト、平均順位などをクリックがなくても取得しますか？ </p> </td>

AMO ID は、インプレッション数、コスト、クリック数、平均順位、平均品質スコアなどの検索エンジン指標を取り込みます。 クリックがなく、インプレッションがある場合は、インプレッション／順位／品質スコアデータが Analytics に送信されます。通常、クリックがない場合は、コストもありません。

+++

+++ このデータはどのレベルで取り込まれますか？ <b>訪問者ですか？ それともヒットですか？</b>

検索エンジン指標は、ヒットレベルでキャプチャされ、AMO ID（およびその分類）に接続されます。 概要レベルデータであり、訪問／訪問者には接続されません。そのため、検索エンジン指標は、ヒットレベルの範囲の、AMO ID（またはその分類）に基づくセグメントにのみ使用できます。

また、AMO ID もランディングページ（訪問／訪問者に接続）のヒットでキャプチャされ、（期限が切れるか新しい AMO ID で上書きされるまで）他の Analytics 指標のクレジットを受け取るためにダウンストリームを保持します。他の任意の eVar のように、データセットに完全に組み込まれます。

+++

+++ google.comのみをキャプチャしますか、または <b>国バージョン</b> (google.co.uk、google.it、google.fr、google.de など ) も同様ですか？

広告プラットフォーム分類は、値「Google Adwords」および「Bing Ads」を取り込みます。 一般的なベストプラクティスとして、キャンペーンの名前の一部に国コードを含めます。これにより、フィルタリングしたりセグメント化したりできます（例：すべてのキャンペーンがcountrycode_ で始まる場合、「UK_」で始まるキャンペーン（AMO ID）のセグメントを作成すると、UK のデータのみが提供されます）。

+++

+++ 「AMO コスト」指標は、検索エンジンでレポートされる、キーワード/広告ごとの有料コストです。 これは、純コストと総コストのどちらですか？

「AMO コスト」は、検索エンジンに支払われたコストのみです。 代理店や検索最適化／管理プラットフォーム代は含まれていません。

+++

+++ 他の広告チャネル ( <b>表示</b> または <b>Social</b>?

いいえ、現在、ロードマップにこれらの他のチャネルの計画はありません。

+++


## 自動トラッキングと手動トラッキング {#section_7437C4698A6D482EB7ED94A948390119}

+++ 自分の Advertising アカウントを設定すると、<b> 自動トラッキング</b> 意図しない結果を招く可能性がある どのような結果が生じますか？

自動モードでは、トラッキングテンプレート/リンク先 URL の末尾に正しい形式で URL パラメーターを追加しようとします。 <b>ただし、ユーザーの責任のもとに、追加された URL パラメーターが最終的なランディングページまで適切に保持されるようにする必要があります。 自動モードではランディング URL にキーワードを挿入できます。ご使用の Web サーバーでは特殊文字を含むキーワードがサポートされない場合があります。

+++

+++ 最初に手動または自動トラッキングを設定した場合、後で他のトラッキングモードに切り替えることはできますか。 何か影響はありますか？

はい、トラッキングモードを切り替えることはできますが、切り替えをおこなう前に、古いトラッキングロジックを削除する必要があります。 これにより、切り替えた日にトラッキングのダウンタイムが発生することがあります（特に手動から自動に切り替えた場合）。そのため、絶対に必要な場合を除き、切り替えをおこなわないことをお勧めします。

* 手動から自動への切り替え：トラッキングテンプレートへの手動追加を削除し、Advertising Analytics UI の切り替えを手動から自動に切り替えて、設定を保存します。 システムが自動トラッキングコードを設定するまでに数時間かかる場合があります。

* 自動から手動への切り替え： Advertising Analytics設定 UI で、切り替えを手動から自動に更新し、できるだけ早く手動のトラッキングコードをデプロイします。 手動トラッキングコードのデプロイ中に、検索エンジンのトラッキングテンプレート内に自動トラッキングコードを見つけた場合は、削除します。
