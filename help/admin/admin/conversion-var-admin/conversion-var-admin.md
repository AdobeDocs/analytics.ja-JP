# コンバージョン変数（eVar）

カスタムインサイトコンバージョン変数（またはeVar）は、サイトの選択したWebページのAdobeコードに配置されます。 その主な目的は、カスタムマーケティングレポートでコンバージョン成功指標をセグメント化することです。 eVarは訪問ベースにでき、cookieと同様に機能します。 eVar変数に渡された値は、事前に決められた期間、ユーザーを追跡します。

eVarを訪問者の値に設定すると、その値が期限切れになるまで自動的に記憶されます。 eVar 値がアクティブなときに訪問者に対して発生したすべての成功イベントは、その eVar 値にカウントされます。

eVarは、次のような原因と効果の測定に最適です。

* 売上に影響を与えた内部キャンペーン
* 最終的に登録に至ったバナー広告
* 注文前に内部検索が使用された回数

トラフィック測定またはパスが必要な場合は、トラフィック変数の使用をお勧めします。

>[!NOTE]イメージリクエストの 1 つの eVar には、単一の値のみを格納できます。1 つの eVar の値に複数の値を格納したい場合は、[リスト変数（リスト var）](https://marketing.adobe.com/resources/help/ja_JP/sc/implement/listN.html)を実装することをお勧めします。

**有効期限**

`eVars` は、指定した期間が過ぎると有効期限切れになります。eVarの有効期限が切れた後は、成功イベントのクレジットを受け取りません。 eVarは、成功イベント時に期限切れになるように設定できます。 例えば、訪問の終了時に期限切れになる内部プロモーションがある場合、その内部プロモーションは、アクティブ化された訪問中に発生した購入または登録に対するクレジットのみを受け取ります。

eVarを期限切れにする方法は2つあります。

* eVarは、指定した期間またはイベント後に有効期限が切れるように設定できます。
* eVarをリセットすることで、eVarの有効期限を強制的に切ることができます。これは、変数を再利用する場合に便利です。

例えば、eVarの有効期限を30日から90日に変更した場合、収集されたeVar値は、新しい有効期限が設定された期間（この場合は90日間）も引き続き保持されます。 システムは、現在の有効期限設定と、収集されたeVar値の最後の設定タイムスタンプを調べて、有効期限を判断します。 このオプション **[!UICONTROL Reset]** のみが値の有効期限が切れ、即座に切れます。

別の例：5月にeVarを使用して内部プロモーションを反映し、21日後に期限切れになり、6月に内部検索キーワードを取り込んだ場合、6月1日に変数の有効期限を強制的に切る（リセットする）必要があります。 これにより、内部プロモーションの値を 6 月のレポートから除外できます。

**大文字と小文字の区別**

eVarでは大文字と小文字が区別されませんが、最初の値は大文字で表示されます。 例えば、eVar1の最初のインスタンスが「ログイン済み」に設定され、それ以降のすべてのインスタンスが「ログイン済み」として渡された場合、レポートには常にeVarの値として「ログイン済み」と表示されます。

**カウンター**

eVarは文字列値を保持するために最もよく使用されますが、カウンターとして機能するように設定することもできます。 eVarは、ユーザーが実行するアクションの数をカウントしてからアクションを実行する場合に、カウンターとして役立ちます。イベントの前に実行するアクションの数は、 例えば、eVarを使用して、購入前の内部検索数を取り込むことができます。 訪問者が検索を行うたびに、eVarに「+1」の値が含まれている必要があります。 訪問者が購入の前に4回検索した場合、合計数ごとに1つのインスタンスが表示されます。1.00、2.00、3.00および4.00。ただし、購入イベント（注文数と売上高指標）のクレジットを受け取るのは4.00のみです。 eVarカウンターの値としては、正の数のみ使用できます。
