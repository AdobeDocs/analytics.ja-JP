# コンバージョン変数（eVar）

カスタムインサイトコンバージョン変数（または eVar）は、サイト上の選択された web ページの Adobe コードに配置されます。 主な目的は、カスタムマーケティングレポートでコンバージョン成功指標をセグメント化することです。 EVarは、訪問ベースで、Cookieと同様に機能します。 eVar 変数に渡された値は、事前設定した期間、ユーザーを追跡します。

**[!UICONTROL Analytics]**／**[!UICONTROL 管理者]**／**[!UICONTROL レポートスイート]**／**[!UICONTROL 設定を編集]**／**[!UICONTROL コンバージョン]**／**[!UICONTROL コンバージョン変数]**

## コンバージョン変数（eVar）の概要

コンバージョン変数の概要については、Analytics チュートリアルガイドの「[&#x200B; コンバージョン変数の概要](https://experienceleague.adobe.com/ja/docs/analytics-learn/tutorials/analysis-workspace/dimensions/introduction-to-conversion-variables-evars)」を参照してください。

EVarが訪問者用の値に設定されると、Adobeは有効期限が切れるまでその値を自動的に記憶します。 eVar 値がアクティブなときに訪問者に対して発生したすべての成功イベントは、その eVar 値にカウントされます。

eVarは、次のような因果関係を測定するのに最適です。

* 収益に影響を与えた社内キャンペーン
* 最終的に登録につながったバナー広告
* 注文を行う前に内部検索が使用された回数

トラフィックの測定またはパスが必要な場合は、トラフィック変数の使用をお勧めします。

>[!NOTE]
>
>イメージリクエストの 1 つの eVar には、単一の値のみを格納できます。 1 つの eVar の値に複数の値を格納したい場合は、[リスト変数（リスト var）](/help/implement/vars/page-vars/page-variables.md)を実装することをお勧めします。

### コンバージョン変数 - 説明 {#section_7C317BB0287A4B8EB0A1A4ECC40627BF}

[コンバージョン変数の編集](/help/admin/tools/manage-rs/edit-settings/conversion-var-admin/conversion-var-admin.md)時に使用されるフィールドの説明。

| 要素 | 説明 |
| --- | --- |
| [!UICONTROL 名前] | コンバージョン変数のわかりやすい名前。 この名前は、通常のレポートで eVar がどのように参照されるかを示しています。またこの名前は、左側のメニューに表示されるレポート／ディメンション名でもあります。 |
| [!UICONTROL タイプ]（eVar のみ） | 次のような変数値のタイプがあります。<ul><li>**[!UICONTROL テキスト文字列]**：サイトで使用しているテキスト値を取得します。 これは、eVarの最も一般的なタイプであり、デフォルト設定です。 他の変数と同様に動作し、その中の値は静的なテキスト文字列です。 内部キャンペーンや内部検索キーワードなどを追跡する場合は、この設定をお勧めします。</li><li>**[!UICONTROL カウンター]**：成功イベントが発生するまでの特定アクションの回数をカウントします。 例えば、eVar を使ってサイト上での内部検索を追跡する場合、この値を「[!UICONTROL テキスト文字列]」に設定して検索語の使用状況を追跡します。 この値を「[!UICONTROL カウンター]」に設定すると、使用された検索語にかかわらず検索回数がカウントされます。 例えば、カウンターeVarを使用して、利用者が購入する前に内部検索を使用した回数を追跡できます。</li></ul> |
| [!UICONTROL 配分] | 成功イベントの前に変数が複数の値を受け取った場合、そのイベントのクレジットを Analytics がどのように割り当てるかを決定します。 次の値がサポートされています。<ul><li>**[!UICONTROL 最新]**：最後の eVar 値が、その eVar の期限が切れるまで、成功イベントのクレジットを常に受け取ります。</li><li>**[!UICONTROL 元の値]**：最初の eVar 値が、その eVar の期限が切れるまで、成功イベントのクレジットを常に受け取ります。</li><li>**[!UICONTROL 線形]**：成功イベントのクレジットを eVar のすべての値に均等に配分します。 線形配分ではある訪問中の値のみが正確に分配されるので、線形配分は訪問の eVar 有効期限を設定して使用します。</li></ul> **メモ**：配分を線形と切り替えると、履歴データが表示されなくなります。 レポートインターフェイスで割り当てタイプを混在すると、レポートで誤ったデータが表示される可能性があります。 たとえば、線形配分では、売上を複数の異なるeVar値に分けることができます。 「直近の配分」に戻ると、その収益の100%が直近の単一の値に関連付けられます。 この関連付けは、ユーザーによる誤った結論につながる可能性があります。<br><br>レポートで混乱が発生する可能性を回避するために、Adobe Analytics では、インターフェイスで履歴データを使用できないようにしています。 指定したeVarを初期割り当て設定に戻す場合は、このオプションを表示できますが、eVar割り当て設定を変更して履歴データにアクセスすることはできません。 Adobeでは、既に大量の履歴データが蓄積されているeVarで配分設定を変更するのではなく、既に記録されているデータに対して新しい配分設定が必要な場合は、新しいeVarを使用することをお勧めします。 |
| [!UICONTROL 有効期限] | EVarの値が期限切れになるまでの期間またはイベントを指定します（サクセスイベントのクレジットは付与されなくなります）。 eVar の有効期限が切れた後に成功イベントが発生した場合、「なし」値がそのイベントのクレジットを受け取ることになります。つまり、アクティブな eVar がないということを示します。  有効期限にイベントを指定した場合、そのイベントが発生した場合にのみ eVar の有効期限が切れます。 イベントが発生しない場合、eVar の有効期限は切れません。  利用可能な期限切れオプションは、主に次の 4 つのカテゴリに分類できます。<ul><li>**ページビューまたは訪問レベルで。** ページビューまたは訪問を超えるコンバージョンイベントは、eVarに関連付けられません。</li><li>**日、週、月、年などの期間に基づきます。** 指定された期間を超えるコンバージョンイベントは、eVarに関連付けられません。 有効期限は、変数が設定されたときに開始されます。 eVarは、設定された時間（分、時間、日、月など）に基づいて期限切れになります。 <ul><li>MINUTE=60秒</li><li>HOUR=3600秒（60分）</li><li>DAY=86400秒（24時間）</li><li>WEEK=604800秒（7日間）</li><li>MONTH=2678400秒（31日）</li><li>QUARTER=8035200秒（93日～31日の3か月）</li><li>YEAR=31536000秒（365日）</li><br>月曜日の午前7:00時に訪問が開始され、その訪問中に午前7:15時にeVarが設定されている場合、有効期限は次のとおりです。<li>日の有効期限：eVarは火曜日の午前7:15時に期限切れになります。</li><li>週の有効期限：eVarの有効期限は、次の月曜日の午前7:15時です。</li><li>月の有効期限：eVarの有効期限は、月曜日の午前7:15時から31日間です。</li></ul><li>**特定のコンバージョンイベント。** EVarに関連付けられて指定された特定のイベントの後に発生するその他のコンバージョンイベント。</li><li>**なし。** 訪問者が同じIDを使用する限り、eVarとイベントの間を任意の時間で通過できます。</li></ul> |
| [!UICONTROL ステータス]（eVar のみ） | [!UICONTROL eVar] のステータスを次のように定義します。<ul><li>**無効**：[!UICONTROL eVar] を無効にします。 [!UICONTROL eVar] をコンバージョン変数リストから削除します。</li><li>**下位関係なし**：[!UICONTROL eVar] をディメンションで分類しないようにします。</li><li>**基本的な下位関係**：任意の完全なディメンション（製品またはキャンペーンなど）で eVar を分類できます。</li></ul> |
| [!UICONTROL リセット] | eVar の既存の値を任意にリセットします。 EVarを再利用する場合は、この設定を使用して、古い値を新しいレポートに混在させないようにします。 リセットしても、履歴データは消去されません。 |

### 有効期限

`eVars` は、指定した期間が過ぎると有効期限切れになります。 EVarの有効期限が切れると、サクセスイベントのクレジットは付与されなくなります。 また、成功イベントで有効期限が切れるようにeVarを設定することもできます。 例えば、訪問の終了時に期限切れとなる内部プロモーションがある場合、内部プロモーションは、その訪問者がアクティブ化された訪問中に発生した購入または登録に対してのみクレジットを受け取ります。

EVarの有効期限を切るには、次の2つの方法があります。

* 指定した期間またはイベントの後に有効期限切れになるように eVar を設定できます。
* eVar をリセットして有効期限切れを強制的に適用することができます。これは、変数を再利用する場合に便利です。

例えば、eVar の有効期限を 30 日から 90 日に変更した場合、収集された eVar 値は、新しい有効期限セット（この場合は 90 日）の間、引き続き保持されます。 システムは単に、現在の有効期限の設定と、収集された eVar 値の最後の設定タイムスタンプを調べて有効期限を判断します。 「**[!UICONTROL リセット]**」オプションを使用した場合にのみ値の有効期限が即座に切れます。

例えば、5 月に内部プロモーションを反映するための eVar を使用して、この eVar の有効期限を 21 日後とし、6 月には内部検索キーワードを取り込むために使用する場合は、6 月 1 日に変数の有効期限切れを強制適用するか、変数をリセットする必要があります。 これにより、内部プロモーションの値を 6 月のレポートから除外できます。

### 大文字と小文字の区別

eVar では大文字と小文字が区別されません。 レポートで使用される大文字と小文字は、バックエンドシステムが登録する最初の値に基づきます。 この値は、レポートスイートに関連付けられているデータの種類や数量に応じて、特定の期間（月など）で確認された初めてのインスタンスになるか、その期間によって変化する可能性があります。

### カウンター

eVarは、文字列値を保持するために最もよく使用されますが、カウンターとして機能するように設定することもできます。 eVarは、ユーザーがイベントの前に実行するアクションの数をカウントする際にカウンターとして便利です。 例えば、eVarを使用して、購入前の内部検索数を取得できます。 訪問者が検索するたびに、eVarには&#39;+1&#39;の値を含める必要があります。 訪問者が購入前に4回検索した場合、各合計数（1.00、2.00、3.00、4.00）のインスタンスが表示されます。 ただし、購入イベント（注文と収益指標）のクレジットを受け取るのは4.00のみです。 EVar カウンターの値として使用できるのは、正の数値のみです。

## コンバージョン変数の追加または編集

1. **[!UICONTROL Analytics]**／**[!UICONTROL 管理者]**／**[!UICONTROL レポートスイート]**&#x200B;の順にクリックします。
1. レポートスイートを選択します。
1. **[!UICONTROL 設定の編集]**／**[!UICONTROL コンバージョン]**／**[!UICONTROL コンバージョン変数]**&#x200B;の順にクリックします。
1. [!UICONTROL コンバージョン変数]ページで、変更するコンバージョン変数の横にある&#x200B;**[!UICONTROL 展開]**&#x200B;アイコン [[+]] をクリックします。

   または

   「**[!UICONTROL 新規追加]**」をクリックして、未使用の eVar をレポートスイートに追加します。
1. 変更するコンバージョン変数フィールドを選択します。

   [&#x200B; コンバージョン変数 – 説明](/help/admin/tools/manage-rs/edit-settings/conversion-var-admin/conversion-var-admin.md#section_7C317BB0287A4B8EB0A1A4ECC40627BF)を参照してください。 フィールドに直接入力できるフィールドもあります。 サポートされている値のドロップダウンリストから選択できます。
1. 「**[!UICONTROL 保存]**」をクリックします。
